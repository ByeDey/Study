# H2 Database

## 개념 요약

**H2**는 Java로 작성된 **경량 인메모리/파일 기반 관계형 데이터베이스**입니다.
별도 설치 없이 의존성만 추가하면 바로 사용할 수 있어 개발 및 테스트 환경에서 자주 사용됩니다.

---

## H2의 특징

- **경량** - 단일 JAR 파일로 동작
- **인메모리 모드** - 애플리케이션 실행 중에만 데이터 유지 (재시작 시 초기화)
- **파일 모드** - 파일에 데이터를 저장해 재시작 후에도 유지
- **웹 콘솔** - 브라우저에서 SQL 실행 및 데이터 확인 가능
- **MySQL 호환 모드** - MySQL 문법과 유사하게 사용 가능
- **빠른 속도** - 인메모리 방식으로 테스트 시 매우 빠름

---

## 모드 종류

| 모드 | URL 형식 | 특징 |
|------|---------|------|
| 인메모리 | `jdbc:h2:mem:testdb` | 재시작 시 데이터 초기화 |
| 파일 | `jdbc:h2:file:./data/chatdb` | 파일에 저장, 재시작 후 유지 |
| 서버 | `jdbc:h2:tcp://localhost/~/testdb` | 외부 접속 가능 |

## Spring Boot에서 H2 설정

**의존성 추가 (pom.xml)**

```xml
<dependency>
    <groupId>com.h2database</groupId>
    <artifactId>h2</artifactId>
    <scope>runtime</scope>
</dependency>
```

**설정 (application.yml)**

```yaml
spring:
  datasource:
    # 파일 모드 (Chat 프로젝트에서 사용)
    url: jdbc:h2:file:./data/chatdb
    driver-class-name: org.h2.Driver
    username: sa
    password:

  h2:
    console:
      enabled: true          # 웹 콘솔 활성화
      path: /h2-console      # 웹 콘솔 접속 경로

  jpa:
    hibernate:
      ddl-auto: update       # 스키마 자동 생성/업데이트
    show-sql: true           # 실행 SQL 로그 출력
```

**ddl-auto 옵션**

| 옵션 | 설명 |
|------|------|
| `create` | 시작 시 테이블 새로 생성 (기존 데이터 삭제) |
| `create-drop` | 시작 시 생성, 종료 시 삭제 |
| `update` | 변경된 스키마만 반영 (기존 데이터 유지) |
| `validate` | 스키마 검증만 수행 |
| `none` | 아무것도 하지 않음 |

## 초기 데이터 설정

**schema.sql** - 테이블 생성 SQL (src/main/resources/)

```sql
CREATE TABLE IF NOT EXISTS room (
    id BIGINT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    creator_id VARCHAR(50),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**data.sql** - 초기 데이터 삽입 SQL (src/main/resources/)

```sql
INSERT INTO room (name, creator_id) VALUES ('일반 채팅방', 'admin');
```

## Chat 프로젝트에서의 H2 활용

Chat 프로젝트에서는 **파일 모드**를 사용하여 서버 재시작 후에도 채팅방과 메시지가 유지되도록 구성했습니다.

```
H2 (파일 모드)               Redis (인메모리)
─────────────────────        ─────────────────────
채팅방 영구 저장              온라인 유저 목록
메시지 영구 저장              채팅방 목록 캐시
참여자 정보 저장              최근 메시지 캐시
                              Rate Limiting 카운터
```

H2에 영구 저장하고, 자주 조회되는 데이터는 Redis에 캐싱하는 구조입니다.

## 주의할 점

- H2는 **개발/테스트 환경**에 적합하며, 운영 환경에서는 MySQL, PostgreSQL 등을 사용해야 한다
- 인메모리 모드는 재시작 시 모든 데이터가 초기화된다
- `ddl-auto: create`는 운영 환경에서 절대 사용하면 안 된다 (데이터 삭제)
- 파일 모드에서 여러 프로세스가 동시에 같은 DB 파일에 접근하면 오류가 발생한다

## 면접 단골 질문

**Q. H2를 개발 환경에서 사용하는 이유는?**
별도 설치 없이 의존성만 추가하면 바로 사용 가능하고, 빠른 속도로 테스트할 수 있어 개발 생산성이 높아지기 때문

**Q. H2 인메모리 모드와 파일 모드의 차이는?**
인메모리 모드는 애플리케이션 실행 중에만 데이터가 유지되고 재시작 시 초기화, 파일 모드는 파일에 저장되어 재시작 후에도 데이터가 유지됨

**Q. `ddl-auto` 옵션 중 운영 환경에서 사용하면 안 되는 것은?**
`create`, `create-drop`은 기존 테이블과 데이터를 삭제하므로 운영 환경에서 절대 사용하면 안 됨
